EVENT FLOW
==========

  kernel syscall fires
       │
       ▼
  eBPF probe captures it ──→ checks eBPF map for existing policy
       │                            │
       │ (no policy)                │ (policy exists)
       ▼                            ▼
  ring buffer ──→ daemon        enforce immediately
       │                        (allow/deny/rewrite)
       ▼
  coordinator triages
       │
       ▼
  spawns child agent goroutine
       │
       ├──→ agent calls Qwen: "what should we do?"
       │         │
       │         ▼
       │    Qwen returns tool_calls
       │         │
       │         ├── read-only?  ──→ execute locally, loop
       │         │
       │         └── write/exec? ──→ send to coordinator queue
       │                                    │
       │                              coordinator checks conflicts
       │                                    │
       │                              approved ──→ execute
       │                              rejected ──→ try alternative
       │
       └──→ task done, goroutine exits
            coordinator updates eBPF map with new policy
            next time this event fires, kernel enforces directly


eBPF CAPABILITIES (all used)
=============================

  OBSERVE     kprobes, tracepoints, uprobes, fentry/fexit, perf_event
  ENFORCE     LSM hooks, seccomp, XDP drop
  TRANSFORM   XDP/TC packet rewrite, socket redirect, DNS rewrite, transparent proxy
  SCHEDULE    sched_ext custom CPU scheduling, per-process priority
  MEASURE     perf counters, in-kernel histograms, per-process metrics
  ITERATE     walk TCP connections, enumerate processes, dump kernel structs
